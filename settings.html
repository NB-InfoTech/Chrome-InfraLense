Transport

HTTPS

HTTPS encrypts traffic between the browser and the website, protecting credentials, cookies, and page data from network interception.

InfraLens includes it because insecure HTTP is one of the clearest signals of transport risk.

Header

HSTS

Strict-Transport-Security tells browsers to use HTTPS for future visits and helps prevent downgrade attacks.

InfraLens checks it because a site can use HTTPS but still be easier to downgrade if HSTS is missing.

Header

CSP

Content-Security-Policy controls which scripts, styles, images, frames, and connections the page is allowed to load.

InfraLens checks it because CSP can reduce the impact of cross-site scripting and unsafe third-party content.

Header

X-Frame-Options

X-Frame-Options helps stop a page from being embedded inside another site without permission.

InfraLens checks it because missing frame protection can expose users to clickjacking attacks.

Header

X-Content-Type-Options

X-Content-Type-Options with nosniff prevents browsers from guessing a file type when the server declared a different one.

InfraLens checks it because MIME sniffing can turn harmless-looking files into executable content.

Header

Referrer-Policy

Referrer-Policy controls how much URL information is sent to other sites when users click links or load resources.

InfraLens checks it because full referrers can leak internal paths, tokens, or sensitive page context.

Header

Permissions-Policy

Permissions-Policy limits browser features such as camera, microphone, geolocation, and payment access.

InfraLens checks it because least-privilege browser capabilities reduce accidental exposure.

Cookies

Cookie Flags

Secure, HttpOnly, and SameSite help protect cookies from network theft, script access, and cross-site request abuse.

InfraLens includes them because session cookies are often the most valuable browser-side asset.

Network

Mixed Content

Mixed content happens when an HTTPS page loads resources over HTTP.

InfraLens checks it because HTTP subresources can weaken an otherwise encrypted page.

Network

Redirect Chain

Redirect chains show the path a browser follows before reaching the final page.

InfraLens includes them because insecure or excessive redirects can hide risk and slow troubleshooting.

Reputation

Sucuri SiteCheck

Sucuri SiteCheck reviews a URL for malware, blocklist status, injected spam, and common website compromise indicators.

InfraLens links to it because reputation checks complement local browser security checks.

Reputation

VirusTotal

VirusTotal aggregates reputation and detection signals from multiple security vendors.

InfraLens links to it because external reputation context is useful during suspicious-site triage.

Privacy

  • InfraLens is read-only.
  • It does not inject scripts into pages.
  • It does not track browsing history or sell data.
  • External check buttons open third-party tools in a new tab only when clicked.
